# 039. Config 카테고리 추가 및 탐색기 구조 개선

## 개요
Wiz 탐색기에서 `Exports` 카테고리를 제거하고, 프로젝트 설정 관리를 위한 `Config` 카테고리를 신설했습니다. 또한 `Project` 카테고리에서 `config` 폴더를 숨기고 별도 그룹으로 분리하여 접근성을 높였습니다.

## 변경 사항

### 1. 카테고리 구조 변경
- **Exports 제거**: 사용 빈도가 낮은 `Exports` 카테고리를 Wiz 탐색기 최상위 목록에서 제거했습니다.
- **Config 추가**: `project/config` 경로를 가리키는 `Config` 카테고리를 추가했습니다.
  - 아이콘: `settings-gear`
  - 컨텍스트 값: `configCategory`

### 2. Project 카테고리 필터링
- `Project` 카테고리 내에서 `config` 폴더가 중복 표시되지 않도록 필터링 로직을 추가했습니다.

### 3. Config 카테고리 기능
- **컨텍스트 메뉴**: 일반 폴더와 동일한 컨텍스트 메뉴(New File, New Folder, Copy, Paste 등)를 사용할 수 있도록 `package.json` 설정을 업데이트했습니다.
- **파일 생성 버그 수정**: Config 카테고리 루트에서 파일 생성 시 `resourceUri` 누락으로 인한 오류를 수정했습니다.

### 4. 버그 수정
- **탐색기 초기화 오류 수정**: 워크스페이스가 로드되지 않은 상태에서 탐색기 초기화 시 발생하는 크래시 문제를 수정했습니다.
